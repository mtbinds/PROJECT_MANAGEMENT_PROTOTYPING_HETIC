# TP : Gestion d’un Projet Informatique - Développement d'une Application CRUD

## Objectifs pédagogiques

- Travailler en équipe et mettre en pratique la gestion de projet informatique.
- Appliquer les concepts de développement d'une application CRUD.
- Utiliser un système de gestion de versions (Git).
- Suivre et gérer un planning de projet avec un **diagramme de Gantt**.
- Livrer un produit fonctionnel dans le respect des contraintes et des délais.

## Description du projet

En groupe de **3 à 5 étudiants**, vous allez développer une application Web ou Desktop permettant de gérer un système de suivi d'un type de données spécifique (par exemple : gestion des étudiants, gestion des tâches, gestion des produits). Cette application doit permettre de **créer, lire, mettre à jour et supprimer** des données.

Vous choisirez l'une des thématiques suivantes :

- Gestion des tâches (To-Do List).
- Gestion des étudiants dans une école/université.
- Gestion des produits dans une boutique en ligne.
- Gestion des livres dans une bibliothèque.
- Gestion des employés dans une entreprise.

Vous devrez vous organiser en tant que groupe pour gérer ce projet, depuis la définition des fonctionnalités jusqu'à la livraison finale de l'application.

## Cahier des charges

### Fonctionnalités principales

L'application doit permettre :

1. **Créer** un nouvel élément (ex : ajouter un étudiant, une tâche, un produit...).
2. **Lire** la liste des éléments existants et afficher les détails d'un élément spécifique.
3. **Mettre à jour** les informations d'un élément (ex : modifier les informations d’un produit).
4. **Supprimer** un élément de la liste.

### Autres exigences fonctionnelles
- **Authentification utilisateur** : Implémentez une page de connexion pour gérer les droits d’accès à l'application.
- **Validation des formulaires** : Les formulaires doivent valider les entrées utilisateur (par exemple, ne pas permettre des champs vides).
- **Design minimal** : Utilisez un framework CSS ou un design personnalisé pour rendre l'interface agréable à utiliser.

### Technologies (au choix selon votre cours)

- Backend : Python (Flask/Django), PHP (Laravel), Java (Spring), Node.js, etc.
- Frontend : HTML/CSS/JavaScript (Bootstrap, Vue.js, React, etc.).
- Base de données : MySQL, PostgreSQL, SQLite, MongoDB, etc.

## Gestion de projet

### 1. Organisation de l'équipe

Chaque membre du groupe doit avoir un rôle spécifique, par exemple :

- **Chef de projet** : Responsable de la coordination générale.
- **Développeur Frontend** : Chargé de l'interface utilisateur.
- **Développeur Backend** : Chargé de la logique serveur.
- **Spécialiste Base de données** : Responsable du modèle de données et de la gestion de la base de données.
- **Testeur** : Responsable de la qualité et des tests fonctionnels.

**Remarque :** Les rôles peuvent être interchangeables, mais il est important de définir clairement les responsabilités.

### 2. Utilisation de Git

Vous devez utiliser **Git** pour gérer votre projet. Chaque membre doit créer et travailler sur une branche spécifique, puis soumettre des pull requests pour intégrer les changements dans la branche principale.

### 3. Planification du projet avec un diagramme de Gantt

Vous devez planifier votre projet en **utilisant un diagramme de Gantt**. Ce diagramme permettra de visualiser les tâches à réaliser, leur durée, les interdépendances, et de suivre l'avancement du projet.

- Le **diagramme de Gantt** doit inclure :

  - Les principales phases du projet (voir ci-dessous).
  - La répartition des tâches entre les membres de l'équipe.
  - Les dates de début et de fin pour chaque tâche.
  - Les jalons importants (livraison des fonctionnalités clés, tests, etc.).

### 4. Étapes du projet à suivre

Le projet devra suivre les étapes suivantes :

- **Phase 1** : Analyse des besoins et rédaction du cahier des charges.
- **Phase 2** : Conception de la base de données et des wireframes de l’interface.
- **Phase 3** : Développement des fonctionnalités CRUD de base.
- **Phase 4** : Ajout de fonctionnalités supplémentaires (authentification, validations).
- **Phase 5** : Tests, corrections de bugs, et livraison finale.

### 5. Outils recommandés

- **GitHub/GitLab/Bitbucket** pour le dépôt de code.
- **Trello** ou **Jira** pour la gestion des tâches.
- **Google Docs** ou **Notion** pour la documentation.
- **Slack** ou **Discord** pour la communication.
- **GanttProject**, **Microsoft Project**, **Asana** ou tout autre outil permettant de créer un **diagramme de Gantt** pour la planification.

## Critères d’évaluation

L’évaluation se fera en prenant en compte les points suivants :

### 1. Fonctionnalités

- Respect du cahier des charges (fonctionnalités CRUD).
- Fonctionnalités supplémentaires (authentification, validation, etc.).

### 2. Gestion de projet

- Respect du planning et organisation du travail en équipe.
- Utilisation correcte de Git (branches, pull requests, commits réguliers).
- Qualité et respect du **diagramme de Gantt** (respect des jalons et des deadlines).

### 3. Qualité du code

- Lisibilité du code, clarté des commentaires et documentation.
- Architecture du projet (respect du modèle MVC ou autre architecture choisie).

### 4. Interface utilisateur

- Simplicité et ergonomie de l’interface.
- Design propre et cohérent.

### 5. Tests et qualité

- Tests fonctionnels (automatisés ou manuels).
- Gestion des erreurs et des cas limites.

### 6. Présentation finale

- Chaque groupe devra présenter son travail en fin de projet (15-20 minutes) et faire une démonstration de l'application.
- Réponses aux questions sur la méthodologie utilisée, les défis rencontrés, et les solutions apportées.

## Livrables

- Code source du projet sur un dépôt Git (GitHub/GitLab/Bitbucket).
- Documentation du projet (fichier README avec instructions de déploiement, documentation technique et utilisateur).
- **Diagramme de Gantt** utilisé pendant le projet.
- Présentation finale du projet.

## Barème

- **Fonctionnalités CRUD** : 30%
- **Gestion de projet et collaboration Git** : 20%
- **Qualité du code et architecture** : 20%
- **Interface utilisateur** : 10%
- **Tests et gestion des erreurs** : 10%
- **Présentation finale et diagramme de Gantt** : 10%

## Deadline

Le projet doit être rendu en fin de journée. Assurez-vous de respecter les délais, car des pénalités de retard peuvent être appliquées.
